Optivan User Manual

Evaluators

An evaluator runs your code or an external solver to turn a design point into numbers. Optivan ships several; they share the same idea — values come in on input pins, a result goes out on an output pin — and each runs in an isolated per-evaluation directory.

Python Script

The workhorse. Two modes:

  • Inline — write a function directly in the node. It exposes a single fan-in input pin; wire all variables into it and name your function’s parameters after them (see Building a Workflow). The return value is the Scalar output.
  • File — point the node at a .py file. Pins map one-to-one to command-line arguments, which is convenient for code you already version-control.

Switch between Inline and File mode in the Properties panel. Inline is fastest to start with.

A Python evaluator can be assigned to a compute target to run on a remote machine or cluster.

Shell Script

Runs any command — a bash one-liner, a compiled binary, a solver. Inputs are passed as environment variables or stdin; outputs are read from stdout or a result file. Use this to wrap anything that isn’t Python or Matlab.

The Shell Script node runs a command in the evaluation’s working directory and reads a scalar result back.

Matlab

Like the Python Script but calls Matlab or Octave, inline or from a file. Assign it to a compute target to pool Matlab licenses on a cluster.

File Mapper

Generates a solver input deck from a template by substituting design variables into placeholders.

  • Wire variables into its fan-in pin.
  • Point it at a template file with placeholders — choose a delimiter style ({thickness}, <thickness>, %thickness%, or [[thickness]]).
  • Map each placeholder to a variable, with an optional number format.
  • Its output is a File pin (the generated deck) you wire into a solver.

The File Mapper fills {thickness} in a template with the current design value and emits the finished deck as a File output.

The classic chain is: Variables → File Mapper → Shell Script (run solver) → Output Parser → Objective.

Output Parser

Extracts a single number from a results file:

  • Regex mode — capture group 1 of a pattern.
  • CSV mode — a column reduced by last, first, min, max, mean, or sum.

Its input is a File; its output is a Scalar you feed to a goal.

Regex or CSV extraction turns a solver’s text output into a scalar for an Objective or Constraint.

For LS-DYNA, you usually skip the Output Parser and read binout / d3plot directly — see LS-DYNA Integration.

Excel

Reads values from an Excel workbook, exposing cells as Scalar outputs — handy when a spreadsheet already encodes part of your model.
